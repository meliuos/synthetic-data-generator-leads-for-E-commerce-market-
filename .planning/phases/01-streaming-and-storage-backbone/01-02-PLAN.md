# Phase 01 Plan 02: ClickHouse 3-Table Schema

## Objective
Implement the Kafka queue table, materialized view, and MergeTree target table in ClickHouse with required event columns and locked ORDER BY.

## Context Files
- @.planning/ROADMAP.md
- @.planning/REQUIREMENTS.md
- @.planning/STATE.md
- @docker-compose.yml

## Tasks

### Task 1: Add SQL schema for analytics ingestion
- type="auto"
- Create idempotent SQL for database bootstrap, Kafka engine source table, MergeTree target table, and materialized view.
- Ensure ORDER BY is `(page_url, event_type, toDate(event_time))`.
- Ensure required columns exist: `x_pct`, `y_pct`, `element_selector`, `device_type`, `session_id`, `anonymous_user_id`.

### Task 2: Add schema apply script and make target
- type="auto"
- Add shell script to apply schema against running ClickHouse container.
- Add Make target to call the script and fail fast on SQL errors.

### Task 3: Verify schema and document query checks
- type="auto"
- Run stack and schema apply command.
- Verify tables exist and ORDER BY expression is correct.
- Document verification query commands in README.

## Verification
- `make validate` exits successfully.
- `make up` followed by `make schema` exits successfully.
- `docker compose exec -T clickhouse clickhouse-client --query "SHOW TABLES FROM analytics"` lists `events_queue`, `click_events`, `events_mv`.

## Success Criteria
1. Three-table ingestion schema exists in ClickHouse and can be recreated safely.
2. MergeTree table stores required event fields and locked ORDER BY.
3. Team has one-command schema application workflow.

## Output
- SQL schema files, script/tooling, and docs committed in atomic task commits.
