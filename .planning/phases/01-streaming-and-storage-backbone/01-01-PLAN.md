# Phase 01 Plan 01: Docker Compose Stack Foundation

## Objective
Stand up a local infrastructure baseline for Phase 1 with Redpanda, ClickHouse, RudderStack data plane, and Streamlit in one Docker Compose stack.

## Context Files
- @.planning/ROADMAP.md
- @.planning/REQUIREMENTS.md
- @.planning/STATE.md

## Tasks

### Task 1: Create project scaffolding for infra and dashboard
- type="auto"
- Create folder layout for infra configs and a minimal Streamlit app container target.
- Add baseline env example and ignore rules for generated runtime data.
- Done when structure is in place and files are documented.

### Task 2: Implement Docker Compose stack and service configs
- type="auto"
- Add compose services for Redpanda, ClickHouse, RudderStack data plane, Postgres (Rudder dependency), and Streamlit.
- Add ClickHouse async insert configuration mount.
- Add RudderStack file-based backend config with a Kafka destination to Redpanda.
- Done when compose file is syntactically valid and references existing local config files.

### Task 3: Add runnable verification workflow for stack bootstrap
- type="auto"
- Add Make targets for validate/up/down/logs.
- Add quickstart documentation with expected health checks.
- Run non-destructive validation command(s).
- Done when contributors can validate config and start stack with one command.

## Verification
- `docker compose config` exits successfully.
- `make validate` exits successfully.

## Success Criteria
1. Repository contains compose + config artifacts for Redpanda, ClickHouse, RudderStack data plane, and Streamlit.
2. ClickHouse async insert settings are explicitly configured.
3. Developers have a single documented bootstrap path for this phase.

## Output
- Infrastructure files and scripts committed in atomic task commits.
- Summary document for 01-01 execution.
