Create a standalone Playwright async service that captures full-page screenshots at desktop (1440px) and mobile (390px) viewports. Screenshots are stored to disk with paths derived from URL hash and viewport, can be requested on-demand via API, and served back through an HTTP endpoint.

Purpose: Enable Phase 4 (Heatmap Dashboard) to load consistent page screenshots for overlay visualization. Viewport-specific captures allow responsive heatmap rendering.

Output: Working screenshot service container running in docker-compose, controllable from dashboard, serving PNG files from predictable filesystem paths.

<task type="auto">
  <name>Create Playwright screenshot service with FastAPI</name>
  <files>services/screenshot/main.py</files>
  <action>
Create a Python service using FastAPI + Playwright that:

1. **Core functionality:**
   - Expose POST /capture endpoint accepting `{url: string}` 
   - Capture full document at 1440px width (desktop) then 390px width (mobile)
   - Use `page.evaluate('document.body.scrollHeight')` after load to capture full height
   - Wait for networkidle before capturing (stability)
   - Return JSON: `{desktop: "/path/1440.png", mobile: "/path/390.png"}`

2. **Path generation:**
   - Hash URL: `hash_object = hashlib.sha256(url.encode()).hexdigest()[:12]`
   - Store at: `/screenshots/{hash}/{viewport}.png` (e.g., `/screenshots/a3f2b1c4d5e6/1440.png`)
   - Create directory tree if missing: `Path(f"/screenshots/{hash}").mkdir(parents=True, exist_ok=True)`

3. **Caching:**
   - Check if `/screenshots/{hash}/{viewport}.png` exists before capturing
   - If exists, skip capture and return path immediately
   - Include `cached: true/false` in response

4. **Error handling:**
   - Return `{error: string, url: string}` on failure (invalid URL, timeout, browser error)
   - Keep error messages descriptive for dashboard debugging
   - 5-second timeout per viewport capture

5. **Health endpoint:**
   - GET /health returns 200 + `{"status": "ready"}`
   - Verify browser process available

Do NOT use Playwright headless_sync or synchronous patterns — must be async (asyncio).
Use `async with async_playwright() as p:` context manager pattern.
  </action>

1. **requirements.txt:** (in services/screenshot/)

   ```
   uvicorn[standard]==0.24.0
   playwright==1.40.0
   pillow==10.1.0
   pydantic==2.5.0
   python-dotenv==1.0.0
   ```

2. **Dockerfile:** (in services/screenshot/)
   ```dockerfile
   FROM python:3.11-slim
   
   WORKDIR /app
   
   # Install system deps for Playwright
   RUN apt-get update && apt-get install -y \
       libnss3 libnspr4 libdbus-1-3 libatk1.0-0 libatk-bridge2.0-0 \
       libcups2 libdrm2 libxkbcommon0 libpango-1.0-0 libcairo2 \
       libxcomposite1 libxdamage1 libxfixes3 libxrandr2 && \
       rm -rf /var/lib/apt/lists/*
   
   # Copy requirements and install Python dependencies
   COPY requirements.txt .
   RUN pip install --no-cache-dir -r requirements.txt
   
   # Install Playwright browsers
   RUN python -m playwright install chromium
   
   # Copy application
   COPY main.py .
   
   # Create screenshots directory
   RUN mkdir -p /screenshots
   
   EXPOSE 8000
   HEALTHCHECK --interval=15s --timeout=5s --retries=3 \
     CMD python -c "import urllib.request; urllib.request.urlopen('http://localhost:8000/health').read()" || exit 1
   
   CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
   ```
