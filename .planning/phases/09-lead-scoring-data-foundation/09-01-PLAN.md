<objective>
Implement Phase 9: Lead Scoring Data Foundation in one plan. Build the four read-time ClickHouse
views that provide the unified event surface and per-session behavioral feature vector for v1.2
lead scoring. No scoring logic in this phase — data layer only.

Deliverables:
1. analytics.unified_events — UNION ALL of click_events + retailrocket_raw.events with a
   normalised event_type vocabulary and a source discriminator column.
2. analytics.live_session_features — GROUP BY session_id on click_events, computing 11 behavioral
   features per session.
3. analytics.retailrocket_session_features — GROUP BY visitor_id+day on retailrocket_raw.events,
   same 11 features with source-specific NULL/zero semantics for fields that don't exist in
   Retailrocket (page_views, search_count, max_scroll_pct).
4. analytics.session_features — UNION ALL of 2 and 3.

Also: verify_features.sql smoke query (7 checks), Makefile targets.
</objective>

## Task 1 — SQL views (004_phase9_foundation.sql)

Write four `CREATE OR REPLACE VIEW` statements in one file:

### analytics.unified_events
UNION ALL of `analytics.click_events` (live) and `retailrocket_raw.events` (dataset).

Common schema:
- `event_id String` — event_id from click_events; CAST(row_hash AS String) from Retailrocket
- `event_time DateTime64(3, 'UTC')` — both sources
- `event_type LowCardinality(String)` — normalised: RR 'view'→'product_view', 'addtocart'→'add_to_cart', 'transaction'→'purchase'
- `page_url Nullable(String)` — from click_events; NULL for Retailrocket (item interactions, not page loads)
- `anonymous_user_id Nullable(String)` — from click_events; toString(visitor_id) for Retailrocket
- `session_id Nullable(String)` — from click_events; concat(toString(visitor_id), '_', toString(toDate(event_time))) for Retailrocket
- `product_id Nullable(String)` — from click_events; toString(item_id) for Retailrocket
- `source String` — 'live' or 'retailrocket'

Type alignment for UNION ALL:
- Use `toNullable(page_url)` on the live side so page_url is Nullable(String) on both sides.
- Use `toNullable(anonymous_user_id)` and `toNullable(session_id)` on the live side for the same reason.
- Use `CAST(NULL AS Nullable(String))` for page_url on the Retailrocket side.

### analytics.live_session_features
GROUP BY (session_id, anonymous_user_id) on analytics.click_events.
Filter: `session_id NOT IN ('', 'unknown_session') AND anonymous_user_id != ''`

Columns:
- session_id String, anonymous_user_id String
- page_views UInt64 = countIf(event_type = 'page_view')
- product_views UInt64 = countIf(event_type = 'product_view')
- add_to_cart_count UInt64 = countIf(event_type = 'add_to_cart')
- purchase_count UInt64 = countIf(event_type = 'purchase')
- search_count UInt64 = countIf(event_type = 'search')
- max_scroll_pct Nullable(Float32) = maxIf(scroll_pct, event_type = 'scroll' AND scroll_pct IS NOT NULL)
  [returns NULL when session has no scroll events — ClickHouse 24.x maxIf(Nullable, cond) behaviour]
- session_duration_seconds UInt32 = toUInt32(greatest(0, dateDiff('second', min(event_time), max(event_time))))
- distinct_products_viewed UInt64 = toUInt64(uniqExactIf(product_id, product_id IS NOT NULL))
- cart_abandoned UInt8 = if(add_to_cart_count > 0 AND purchase_count = 0, 1, 0)
- first_event_at DateTime64(3, 'UTC') = min(event_time)
- last_event_at DateTime64(3, 'UTC') = max(event_time)
- source String = 'live'

### analytics.retailrocket_session_features
GROUP BY synthesised session key on retailrocket_raw.events.
Session key = concat(toString(visitor_id), '_', toString(toDate(event_time))).
Use WITH clause: `WITH concat(...) AS session_id, toString(visitor_id) AS anonymous_user_id`

Same columns as live_session_features, with these differences:
- page_views = toUInt64(0)  [no page_view events in Retailrocket]
- product_views = countIf(event_type = 'view')  [Retailrocket vocab]
- add_to_cart_count = countIf(event_type = 'addtocart')
- purchase_count = countIf(event_type = 'transaction')
- search_count = toUInt64(0)  [no search events]
- max_scroll_pct = CAST(NULL AS Nullable(Float32))  [no scroll data]
- distinct_products_viewed = toUInt64(uniqExact(item_id))  [item_id is UInt64, non-nullable]
- cart_abandoned = if(countIf(event_type = 'addtocart') > 0 AND countIf(event_type = 'transaction') = 0, 1, 0)
- source = 'retailrocket'

### analytics.session_features
```sql
CREATE OR REPLACE VIEW analytics.session_features AS
SELECT * FROM analytics.live_session_features
UNION ALL
SELECT * FROM analytics.retailrocket_session_features;
```

## Task 2 — Smoke query (scripts/verify_features.sql)

Seven verification queries (named with check_name column for readability):
1. unified_events row count by source (expect both > 0)
2. unified_events Retailrocket normalised vocab (expect product_view/add_to_cart/purchase, NOT view/addtocart/transaction)
3. session_features row count by source
4. live_session_features feature completeness (counts of non-zero features)
5. retailrocket_session_features invariants (page_views=0 always, max_scroll_pct=NULL always)
6. Cross-source feature averages (avg product_views, add_to_cart, purchase, duration)
7. Product ID coverage for e-commerce events by source
