# Phase 9: Rule-based Lead Intelligence Dashboard

**Goal**: Implement a rule-based lead scoring engine natively in ClickHouse using the Retailrocket dataset, and expose an interactive "Lead Intelligence" dashboard panel in Streamlit that ranks high-intent users and flags cart abandoners.
**Requirements covered**: LEAD-01, LEAD-02

## Pre-flight Checklist

- [x] Verify Phase 7 Retailrocket data is successfully imported (`retailrocket_raw.events` is populated).
- [x] Verify ClickHouse can perform aggregation queries on millions of rows in < 2 seconds.

## Execution Steps

1. **ClickHouse Query Layer**: Create `dashboard/lead_queries.py`.
   - Implement `fetch_top_leads` using a heuristic score: `(views * 1) + (add_to_carts * 20) + (purchases * 100)`.
   - Implement `fetch_lead_summary` to aggregate total visitors, qualified leads, and cart abandoners.
   - Execute all logic natively in ClickHouse to avoid fetching raw rows into Python.

2. **Streamlit UI Integration**: Modify `dashboard/app.py`.
   - Wrap the existing v1.0/v1.1 Heatmap logic into a dedicated "Heatmap & Stats" tab.
   - Create a "Lead Intelligence (Synthetic)" tab.
   - Add a threshold slider (1-500 points) to dynamically filter leads.
   - Render the summary metrics and the interactive `st.dataframe` showing the top 100 leads, highlighting cart abandoners in red.

3. **Validation**: 
   - Verify the Streamlit app loads without performance degradation.
   - Ensure the points calculation matches the SQL heuristic and cart abandoners correctly reflect `add_to_carts > 0` and `purchases = 0`.
