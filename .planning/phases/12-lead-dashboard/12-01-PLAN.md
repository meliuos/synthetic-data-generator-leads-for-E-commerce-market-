# Phase 12: Lead Identification Dashboard

## Goal
Surface the downstream output of both the rule-based and ML scoring pipelines (from Phases 10 and 11) directly inside the Streamlit UI, allowing marketers/sales reps to visualize their lead funnel and inspect highly-scored candidate sessions.

## User Stories / Tasks

### User Story 1: ClickHouse Query Layer (`dashboard/heatmap_queries.py`)
*As a developer, I need to fetch the scored lead data efficiently from ClickHouse to power the visualizations.*
- [x] Write an aggregate SQL method that extracts the proportional volume of `hot`, `warm`, and `cold` scored sessions.
- [x] Write a SQL method that `JOIN`s `analytics.lead_scores_ml` descending on the best ML scores with their respective metrics from `analytics.lead_scores_rule_based` and limits to the Top `N` sessions.

### User Story 2: Dashboard UI Scaffolding (`dashboard/app.py`)
*As a marketer, I want a dedicated sub-view in the analytics platform so that the 'heatmap analytics' and 'sales leads intelligence' metrics are separated.*
- [x] Migrate the existing Streamlit layout to feature distinct interface tabs (e.g., `st.tabs(["Heatmap & Analytics", "Sales Leads"])`).
- [x] Inside the Leads tab, render a graphical distribution chart (e.g., Pie or Bar using Plotly) of the `score_tier` aggregation.
- [x] Render a table representing the `Top N Ranked Leads` containing discrete ML Model Percentage (`ml_score_pct`) along with their Source & Associated IDs.

### User Story 3: Score Drill-down Detail View 
*As a sales rep, I need to understand why a specific candidate is 'HOT' so that I can personalize my interactions based on their funnel behavior.*
- [x] Implement row selection or a pop-up expander directly attached to the Streamlit top-leads table. 
- [x] Ensure the click event retrieves the specific lead's raw features and `rule_contributions` JSON so they can be read in plain-text (e.g., showing if they "searched multiple times", "abandoned a cart", read "3 different items", etc.).
