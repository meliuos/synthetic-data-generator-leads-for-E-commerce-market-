# Phase 7: Retailrocket Import

**Goal**: The Retailrocket open-source dataset (events, item_properties, category_tree) loads idempotently into a parallel `retailrocket_raw.*` table set.
**Requirements covered**: DATA-01, DATA-02, DATA-03, DATA-04, DATA-05, DATA-06

## Pre-flight Checklist

- [x] Kaggle License block screenshot committed under `.planning/research/v1.1/evidence/kaggle-license.png`
- [x] Verify `~/.kaggle/kaggle.json` exists locally for authentication

## Execution Steps

1. **Setup Environment**: `make retailrocket-setup` creates `.venv-retailrocket` and installs `kaggle` CLI + `clickhouse-connect`.
2. **Download Data**: `make retailrocket-download` uses the Kaggle API to fetch `ecommerce-dataset` and extracts the 4 required CSV files into `data/retailrocket/`.
3. **Apply Schema**: `make schema-retailrocket` ensures the `retailrocket_raw` database and tables (`events`, `item_properties`, `category_tree`, `item_latest` view) are present.
4. **Import Data**: `make retailrocket-import` reads the CSVs in 500k chunks, computing row hashes and using ClickHouse's `insert_deduplication_token` to load them idempotently.
5. **Verify Idempotency**: Run `make retailrocket-import` a second time to ensure it instantly exits without duplicating rows.
6. **Smoke Test**: `make retailrocket-smoke` verifies total row counts and event distributions against source expectations.
